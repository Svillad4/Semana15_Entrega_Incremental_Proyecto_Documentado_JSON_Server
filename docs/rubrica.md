# Rubrica de evaluacion

| Criterio | Porcentaje | Nivel esperado |
|---|---:|---|
| Funcionalidad incremental demostrable | 25% | El proyecto ejecuta, guarda datos, muestra bitacora, checklist y dashboard. |
| Integracion correcta con JSON Server | 15% | La API local funciona y los datos se consultan o guardan correctamente. |
| Validaciones operativas | 15% | Los formularios validan campos obligatorios y formatos. |
| Documentacion tecnica completa | 20% | El README de entrega explica instalacion, ejecucion, estructura y pruebas. |
| Bitacora y checklist de requisitos | 15% | Hay pruebas, errores, correcciones, estados y evidencias claras. |
| Calidad visual, usabilidad y organizacion | 10% | El proyecto es claro, responsive y esta bien organizado. |

## Niveles de desempeno

### Superior

Entrega completa, funcional, documentada, con evidencias claras y exposicion preparada.

### Alto

Entrega funcional y documentada, aunque puede mejorar la profundidad de evidencias o explicaciones.

### Basico

Entrega parcial; el proyecto funciona en parte, pero faltan documentos, pruebas o evidencias.

### Bajo

No demuestra funcionamiento suficiente, documentacion incompleta o ausencia de pruebas.
