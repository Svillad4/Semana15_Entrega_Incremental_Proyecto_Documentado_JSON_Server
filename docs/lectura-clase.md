# Lectura de clase - Semana 15

## Documentacion tecnica, bitacora de validacion y preparacion de exposicion

En esta semana trabajas una parte fundamental del desarrollo de software: entregar un proyecto de forma clara, verificable y explicable. Un proyecto no termina cuando el codigo funciona una vez en tu computador. Un proyecto esta mejor preparado cuando otra persona puede abrirlo, instalarlo, ejecutarlo, revisar sus pruebas y comprender como fue construido.

La documentacion tecnica sirve para explicar el proyecto. La bitacora de validacion y correccion sirve para demostrar que hiciste pruebas, encontraste errores y aplicaste mejoras. La exposicion tecnica sirve para comunicar el resultado de forma ordenada.

## 1. Documentacion tecnica

La documentacion tecnica es la explicacion escrita que acompana un proyecto. En este taller se trabaja principalmente en `README_ENTREGA.md`. Ese archivo debe responder preguntas como:

- Que hace el proyecto?
- Para que fue creado?
- Que tecnologias utiliza?
- Como se instala?
- Como se ejecuta?
- Como esta organizado?
- Que funcionalidades tiene?
- Que pruebas se realizaron?
- Que errores se corrigieron?

Una buena documentacion evita que el proyecto dependa solamente de la memoria del estudiante.

## 2. Bitacora de validacion y correccion

La bitacora es un registro tecnico. Alli se anota que se probo, que se esperaba obtener, que ocurrio realmente, que error aparecio y que correccion se aplico.

Ejemplo:

| Prueba | Esperado | Obtenido | Correccion |
|---|---|---|---|
| Enviar formulario vacio | Mostrar error | El sistema permitia enviar | Se agrego validacion de campos obligatorios |

Una bitacora demuestra proceso. No solo muestra el resultado final, tambien muestra como llegaste a el.

## 3. Preparacion de exposicion

La exposicion tecnica no consiste en leer todo el codigo. Consiste en explicar de forma breve:

1. Nombre del proyecto.
2. Problema o necesidad.
3. Solucion desarrollada.
4. Tecnologias utilizadas.
5. Funcionamiento principal.
6. Pruebas realizadas.
7. Correcciones aplicadas.
8. Aprendizaje final.

La mejor exposicion es demostrativa: abre el proyecto, muestra una accion funcionando y explica que evidencia demuestra que la prueba fue correcta.
